## Правила безопасности

- Не коммитить секреты, API keys, Bearer tokens, Railway tokens, n8n API keys.
- В документации писать только имена переменных окружения, не их значения.
- Все production-изменения делать через read-only проверку, затем patch/preview, затем apply после подтверждения.
